# Plan 5.6: Real-time WebSocket Streaming

## Objective
Build a lightweight, asyncio-driven WebSocket service to stream live Binance Klines, updating the latest price in real-time and saving closed candles directly to the PostgreSQL database.

## Context
- backend/market_data/normalizers/binance.py
- backend/database.py

## Tasks

<task type="auto" effort="medium">
  <name>Extend Normalizer for WebSockets</name>
  <action>
    Add a new method `normalize_ws_kline` to `BinanceNormalizer` in `backend/market_data/normalizers/binance.py`.
    It must parse the standard `{ "e": "kline", "k": {...} }` payload from Binance WS and return a `NormalizedCandle` alongside its `is_closed` status.
  </action>
  <verify>grep "normalize_ws_kline" backend/market_data/normalizers/binance.py</verify>
  <done>Normalizer supports both REST arrays and WS JSON objects.</done>
</task>

<task type="auto" effort="high">
  <name>Build WebSocket Client</name>
  <action>
    Create `backend/market_data/websocket/binance_ws.py`.
    Use `websockets` and `asyncio` to connect to `wss://stream.binance.com:9443/ws/btcusdt@kline_1m/ethusdt@kline_1m`.
    Implement an in-memory cache for the latest price (avoiding Redis).
    When a candle closes (`x: true`), use SQLAlchemy to store it cleanly in the DB via the `store_candles` logic.
  </action>
  <verify>grep "websockets.connect" backend/market_data/websocket/binance_ws.py</verify>
  <done>Websocket streams are active and safely handling database bounds.</done>
</task>

## Success Criteria
- [ ] No microservices, Kafka, or Redis are introduced.
- [ ] Live price tracks flawlessly in memory.
- [ ] Completed candles are automatically archived into historical storage.
