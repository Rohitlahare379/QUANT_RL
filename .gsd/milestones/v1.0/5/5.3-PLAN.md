# Plan 5.3: Binance Historical Ingestion

## Objective
Build a robust, reusable historical candle ingestion service that fetches, normalizes, and safely stores Binance Klines into the Aegis PostgreSQL database.

## Context
- backend/market_data/storage/models.py
- backend/database.py

## Tasks

<task type="auto" effort="high">
  <name>Build Binance Ingestion Service</name>
  <action>
    Create `backend/market_data/ingestion/fetch_binance.py`.
    Implement `BinanceIngester` class.
    Features:
    - Uses `httpx` to fetch from `https://api.binance.com/api/v3/klines`.
    - Handles pagination using `startTime` and `endTime`.
    - Normalizes raw arrays into `pandas` DataFrames.
    - Uses SQLAlchemy PostgreSQL dialact `insert(...).on_conflict_do_nothing()` for bulletproof bulk upserts without duplicate errors.
    - Supports BTCUSDT, ETHUSDT across 1m, 5m, 1h.
    - Integrates standard Python `logging`.
  </action>
  <verify>grep "BinanceIngester" backend/market_data/ingestion/fetch_binance.py</verify>
  <done>Service written with pagination and UPSERT logic.</done>
</task>
